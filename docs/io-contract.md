# Input Output Contract

This is the exact folder contract expected by UNav mapping scripts.

## Parameter Definitions

Command format:

```bash
python -m unav.run_mapper <data_temp_root> <data_final_root> <feature_model> <place> <building> <floor>
```

- `data_temp_root`: input raw map files + intermediate artifacts
- `data_final_root`: final map artifacts for localization/navigation
- `feature_model`: `DinoV2Salad` | `MixVPR` | `CricaVPR` | `NetVlad` | `AnyLoc`
- `place` / `building` / `floor`: namespace identifiers

## Required Inputs

### Under `data_temp_root`

```text
<data_temp_root>/
├── orb_vocab.fbow
└── <place>/<building>/<floor>/
    └── <floor>.mp4
```

### Under `data_final_root` (needed for aligner)

```text
<data_final_root>/
├── scale.json
└── <place>/<building>/<floor>/
    └── floorplan.png
```

## Generated Outputs

### Intermediate (`data_temp_root`)

```text
<data_temp_root>/<place>/<building>/<floor>/
├── stella_vslam_dense/
│   ├── final_map.msg
│   ├── output_cloud.ply
│   ├── keyframes/
│   └── eval_logs/keyframe_trajectory.txt
├── perspectives/*.png
└── colmap_sfm/
    ├── sparse/0/cameras.txt
    ├── sparse/0/images.txt
    ├── pairs.txt
    ├── matches.h5
    └── database.db
```

### Final (`data_final_root`)

```text
<data_final_root>/<place>/<building>/<floor>/
├── colmap_map/                  # triangulated map output
├── features/
│   ├── local_features.h5
│   └── global_features_<model>.h5
├── floorplan.png
├── transform_matrix.npy         # generated after aligner step
└── boundaries.json              # generated by labelme (map labeling step)
```

## Minimal Deliverable for Navigation

For each floor, ensure these files exist:

- `floorplan.png`
- `boundaries.json`
- `transform_matrix.npy`

For multilingual labels, also provide:

- `_i18n/labels.json` under `data_final_root`
